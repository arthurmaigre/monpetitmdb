## 3. Points techniques résiduels à corriger

### P1 — Images sans next/image (impact LCP)

La homepage utilise `next/image` mais certains composants de listing pourraient utiliser `<img>` natif. Impact :
- Pas de WebP/AVIF automatique → images 30-50% plus lourdes
- Pas de lazy loading natif optimisé
- Pas de srcset responsive

**Action :** vérifier les composants `BienCard`, `EnchereCard` et tout composant visible sur les pages publiques. Migrer `<img>` → `<Image width={...} height={...}>` avec `priority` sur la première image visible.

**Attention :** `next.config.ts` a `remotePatterns: [{ protocol: 'https', hostname: '**' }]` → toutes les sources HTTPS sont déjà autorisées ✅

### P2 — Page `/faq` en 'use client'

Le JSON-LD FAQPage est rendu côté client. Google exécute JavaScript donc il sera indexé, mais SSR serait préférable pour la fiabilité. Solution à terme : séparer le JSON-LD dans un Server Component wrapper. Non urgent.

### P3 — Duplicate dans les FAQ existantes

"Puis-je annuler mon abonnement ?" apparaît deux fois (indices 6 et 11). Ne casse rien mais est sous-optimal pour l'UX et le JSON-LD.
